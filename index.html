Difference between Object.freeze(), Object.seal() and Object.preventExtensions()

The Object.freeze() method

  • This method completely freezes an object.
  • It means that if an object is frozen, then we can’t add a new property to it,
  • can’t update any existing property value,
  • and we also can’t delete any existing properties.
  • To check if an object is frozen, we can use the Object.isFrozen() method.

  • How to thaw/defrost a frozen object in Javascript?

    1. Nothing can be added to or removed from the properties set of a frozen object. Any attempt to do so will fail, either silently or by throwing a TypeError exception (most commonly, but not exclusively, when in strict mode).
    2. Values cannot be changed for data properties. Accessor properties (getters and setters) work the same (and still give the illusion that you are changing the value). Note that values that are objects can still be modified, unless they are also frozen.
    3. So, the only way this could be done is, by cloning the object.

The Object.seal() Method

  • Object.seal() Method seals an object. It means that if an object is sealed, then we can only update the value of existing properties, but we can neither delete any property nor add a new property.

  • To check if an object is sealed, we can use the Object.isSealed() method.

The Object.preventExtensions() Method

  • Object.preventExtensions() method prevents new properties from being added to an object but lets the existing property be updated or deleted.
  • To check if an object is prevented from extensions, we can use the Object.isExtensible() method.

Using strict mode

  • When we use any three of the above methods on an object and later try to mutate it, it doesn’t work, but we also don’t get any console errors.
  • When we use strict mode, it throws an error in such cases.

Working with nested objects

  • The freeze(), seal(), and preventExtensions() methods apply only to the first level of properties.
  • If any nested objects are present in an object, then modifying the nested objects works as before.
  • So to apply these methods to all nested objects, we need to make recursive calls and apply these methods to each level of objects individually.
